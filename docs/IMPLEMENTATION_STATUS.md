# Текущий статус реализации

Обновлено: 20 июля 2026 года.

## Реализованный функциональный MVP

- Next.js 16 App Router, React 19, TypeScript, Tailwind CSS 4 и shadcn/ui.
- Адаптивная UI-система с тёплой светлой и графитовой тёмной темой,
  сохранением выбора в `localStorage`, активной desktop-навигацией и мобильной
  нижней панелью.
- Семейные профили без паролей: создание по имени, выбор через `localStorage`, постоянный индикатор и смена профиля.
- Placement-тест из 15 вопросов с исходными порогами и ручной выбор A1–C1.
- Полный банк прототипа разделён на типизированный каталог по уровням и видам,
  проходит Zod/runtime-проверку и загружается идемпотентным Prisma seed: 292
  записи.
- A1 является утверждённым эталоном глубины: шесть структурированных уроков,
  расширенные grammar/vocabulary/conjugation карточки, вопросы к чтению и
  аудированию и разборы обоих направлений перевода отображаются в приложении.
- Один стабильный `DailyPlan` на профиль и локальную дату `Europe/Madrid`.
- Адаптивный выбор по уровню ±1, слабым темам и недавним ID с лимитами 40/30/15.
- 10 MCQ, 10 fill, 10 слов, доступные спряжения (до 20), чтение, 1–2 аудио и два перевода.
- Теория дня: слабая пройденная тема ниже 70%, затем новая, затем случайная; пересекающиеся попытки общие.
- Серверная фиксация неизменяемой первой попытки и восстановление состояния после reload.
- Completion полного дня, итоговая точность и идемпотентное обновление streak.
- Карточки практики сохраняют введённый ответ после проверки; контекст слова
  виден до ответа, а подсказка о времени в fill-in раскрывается по желанию.
- Теория дня дополнена раскрываемыми правилами и примерами из связанных
  упражнений; общие с практикой попытки явно обозначены.
- Анализ дня разделяет практическую грамматику и закрепление теории, считает
  целые карточки и показывает правила с примерами по слабым темам.
- История содержит календарь и детали дня, а прогресс — календарь, группы тем,
  статистику слов и спряжений и явный переход к смене уровня.
- Изменения прогресса и завершение дня выполняются в сериализуемых транзакциях;
  streak обновляется в шапке сразу после завершения.
- Prisma 7 + PostgreSQL 17, health endpoint и локальный development compose.
- Production Dockerfile с отдельным migrator target и production compose для прямого LAN-доступа.
- Production compose публикует Next.js на host-порту `8888` NAS (`APP_PORT` позволяет переопределить его), сохраняя внутренний порт `3000` для healthcheck.
- Production build использует официальный Webpack fallback: повторный Turbopack build зависал в локальной среде; development остаётся на Turbopack.

## Проверено локально

- Обе миграции применены к PostgreSQL 17.
- `content:check` подтверждает 292 уникальные записи, 73 grammar-темы,
  минимальные объёмы, ссылки, ответы, отсутствие orphan-тем, словарные
  `contextTarget`, multi-blank примеры, полный A1–C1-шаблон, двусторонние
  `lessonIds`/`exerciseIds` и manifest защищённых полей.
- Seed повторно загрузил те же 292 записи контента в существующую базу:
  контрольная сумма таблицы `ContentItem` после второго прогона не изменилась.
- TypeScript, ESLint, Prettier и unit-тесты доменных правил проходят.
- Production build и Docker Compose config входят в обязательный `npm run verify` / `npm run docker:config`.
- Browser acceptance выполняется после каждого функционального изменения; актуальный результат фиксируется в handoff/commit.

## Осознанно остаётся до production-ввода

- интеграционные тесты изоляции двух профилей и конкурентных first-try запросов;
- фактический backup/restore прогон на NAS;
- deployment и проверка рестарта Synology.

Это уже рабочее приложение, но не утверждение о готовом NAS deployment. Исходный `temp/` остаётся локальным эталоном и не коммитится.
