# Документация Español Diario

Этот каталог — основной контракт проекта. Исходный HTML остаётся эталоном поведения до достижения полного функционального паритета новой реализацией.

## Порядок чтения

1. [PRODUCT.md](PRODUCT.md) — зачем существует приложение, кто им пользуется и что входит в первую версию.
2. [CURRENT_PROTOTYPE.md](CURRENT_PROTOTYPE.md) — что именно делает текущий HTML.
3. [TARGET_ARCHITECTURE.md](TARGET_ARCHITECTURE.md) — как это будет устроено на Next.js, Prisma и PostgreSQL.
4. [DATA_MODEL.md](DATA_MODEL.md) — сущности, связи и правила хранения данных.
5. [MIGRATION_PLAN.md](MIGRATION_PLAN.md) — последовательность реализации и критерии функционального паритета.
6. [DEPLOYMENT.md](DEPLOYMENT.md) — контейнеры, PostgreSQL и эксплуатация на Synology.
7. [OPEN_DECISIONS.md](OPEN_DECISIONS.md) — вопросы, которые пока намеренно не зафиксированы окончательно.
8. [IMPLEMENTATION_STATUS.md](IMPLEMENTATION_STATUS.md) — что уже собрано и какой срез рекомендуется следующим.
9. [CONTENT_EXPANSION_PLAN.md](CONTENT_EXPANSION_PLAN.md) — аудит seed-контента, целевая структура каталога и план его расширения.
10. [UI_UX.md](UI_UX.md) — визуальная система, адаптивность, темы и границы параллельной работы с контентом.

## Источники истины

При расхождении документов используется следующий приоритет:

1. Для существующего поведения — `temp/espanol-diario.html`.
2. Для продуктовых границ и обязательного паритета — `PRODUCT.md` и `MIGRATION_PLAN.md`.
3. Для технической реализации — `TARGET_ARCHITECTURE.md` и `DATA_MODEL.md`.
4. Для эксплуатации — `DEPLOYMENT.md`.

Если при реализации обнаружено неоднозначное или ошибочное поведение прототипа, его нельзя молча «улучшить». Нужно зафиксировать расхождение в `OPEN_DECISIONS.md`, принять решение и только затем менять контракт.

## Термины

- **Профиль** — член семьи и отдельный владелец прогресса. Это не защищённая учётная запись.
- **Уровень** — CEFR-уровень A1, A2, B1, B2 или C1.
- **День занятий** — один зафиксированный набор упражнений для профиля и локальной календарной даты.
- **Попытка** — первая отправка ответа на конкретное задание дня; именно она влияет на статистику.
- **Контент** — грамматика, теория, слова, спряжения, чтение, переводы и ссылки на аудирование.
- **Функциональный паритет** — пользователь может выполнить тот же сценарий и получить тот же результат, что в HTML-прототипе, даже если внутренняя реализация другая.
